# Phase 11 Plan 01 — Game Boy emulator dogfood (vertical slice)

## Objective

Execute **DOGFOOD-01**: one **external** git repository (Game Boy emulator minimal slice) progressed by Kiln from spec through at least **coding + testing** stages with **deterministic** scenario commands, using **only** open test artifacts. Prefer **merged** or **failed with typed reason** as a clear terminal state.

## Preconditions

- Phase **10** UAT passed for the operator machine.
- Throwaway GitHub repo created; `GH_TOKEN` (if PR path) scoped minimally.
- Spike decisions in [GB-SPIKE.md](../999.2-gameboy-emulator-dogfood/GB-SPIKE.md) resolved (language, workflow template, sandbox image).

## Falsifiable UAT

1. **Spec + scenarios** — Kiln holds a spec (markdown + embedded BDD) that references the external repo layout; at least **three** `Given/When/Then` scenarios map to shell commands (`cargo test …` or equivalent).
2. **Start run** — From LiveView, operator starts a run bound to the workflow + repo; run appears on board within 10s.
3. **Progress** — Run reaches `coding` or `testing` with visible stage logs / audit events (no silent stall past bounded timeout).
4. **Oracle** — Verifier stage exit status matches scenario runner (pass ⇒ continue or merge per workflow; fail ⇒ loop or **typed** escalation, never ambiguous hang).
5. **Cost / safety** — Run respects configured caps (`ORCH-06`); no secrets in sandbox env (`SAND-*`, `SEC-01`).

## Tasks (high level)

1. Scaffold external repo (Rust or chosen stack); commit minimal CPU test harness + CI.
2. Author Kiln workflow YAML + spec in Kiln (or intake path); wire model profile + caps.
3. Configure workspace git remote to throwaway repo; validate `git push` / PR if in scope.
4. Run once; capture postmortem notes in `11-01-SUMMARY.md` (created at execution).

## Out of scope

- Full emulator compatibility; audio; commercial games.
- **999.3** Docker DX implementation.
