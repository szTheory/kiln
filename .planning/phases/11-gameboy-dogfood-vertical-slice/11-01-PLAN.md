<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Spec author → scenario IR | IR is validated by JSV before compile |
| Compiled tests → host OS | `shell` steps run on developer/CI host during `mix test` — **must not** interpolate unchecked strings into `/bin/sh -c` |
| Operator → throwaway GitHub | Tokens never logged; workspace paths constrained by existing `PushWorker` rules |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | T | `shell` argv | mitigate | **Argv array only** — compiler emits `System.cmd(program, args, opts)` with no shell; reject `argv` containing `";"` or newlines at validation if needed |
| T-11-02 | E | dogfood spec in `priv/dogfood` | mitigate | Spec is static template bytes; no secrets |
| T-11-03 | I | scenario manifest hash | mitigate | Existing `manifest_sha256` covers IR bytes including new step kinds |

## ASVS L1 Notes

Shell oracle bridge is **local test execution** only — cap argv length in schema (e.g. each string ≤ 500 chars, max 32 args) to limit abuse surface in authored specs.
</threat_model>
