<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| same-repo persistence -> advisory evaluator | Candidate reads must stay bounded to one attached repo so warnings do not leak across repos. |
| GitHub CLI -> preflight findings | Open-PR data comes from external CLI output and must be normalized before shaping warnings or fatal ambiguity findings. |
| attach advisory seam -> runs start seam | Advisory findings may influence UX, but deterministic run-start authority must remain unchanged. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-04 | I | same-repo candidate queries | mitigate | Filter every candidate query by `attached_repo_id` and use bounded limits so cross-repo data never appears in findings. |
| T-34-05 | T | open-PR overlap lookup | mitigate | Query `gh pr list` through the existing CLI runner seam, parse only needed JSON fields, and tie findings to repo/base/head evidence. |
| T-34-06 | E | run-start authority | mitigate | Keep fuzzy overlap warnings out of `Runs.start_for_attached_request/3`; preserve deterministic blocked/error returns for operator setup and provider secrets only. |

</threat_model>
