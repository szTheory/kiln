<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deterministic safety gate -> advisory preflight | Hard blockers and fuzzy warnings must stay separated so unsafe mutation checks do not drift into heuristic logic. |
| server-owned attach context -> report | Brownfield findings must be computed from attached-repo identity and durable same-repo records, not browser-local state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-34-01 | T | `Kiln.Attach.BrownfieldPreflight` contract | mitigate | Model severities and evidence explicitly so warning findings cannot masquerade as fatal blockers. |
| T-34-02 | S | attach advisory boundary | mitigate | Expose the preflight only through `Kiln.Attach` with attached-repo ids and bounded request params; do not trust LiveView-computed state. |
| T-34-03 | R | brownfield evidence | mitigate | Require durable ids, repo slug, branch, or snapshot references in findings so later UI can explain why a warning exists. |

</threat_model>
