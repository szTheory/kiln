Create **`lib/kiln/runs/fair_round_robin.ex`**:

**`@spec order([Run.t()], String.t() | nil) :: [Run.t()]`**

Algorithm (document in `@moduledoc`):

1. **Stable sort** input by **`{inserted_at asc, id asc}`** (use **`Enum.sort_by/2`** with tuple comparator `{r.inserted_at, r.id}`).
2. If **`cursor == nil`**, return sorted list as-is (degenerate RR = FIFO by D-01 tie-break).
3. If **`cursor`** is set, find **index** of the run whose **`id == cursor`** in the sorted list. If **not found** (cursor stale), treat as **`nil`**.
4. **Rotate** sorted list so the **next** run after cursor is first: **`Enum.drop(list, idx + 1) ++ Enum.take(list, idx + 1)`** — verify off-by-one on paper with 3 fake UUIDs.

**Tests** in **`test/kiln/runs/fair_round_robin_test.exs`**: table cases for N=1..4 runs, assert **first element** id after `order/2` matches expected RR advance when cursor given.

Use **`%Run{id: ..., inserted_at: ...}`** structs built with **`struct!(Run, ...)`** or minimal changesets — **no** DB unless easier via factory.
  </action>
