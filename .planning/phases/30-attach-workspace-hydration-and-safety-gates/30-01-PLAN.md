<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator input -> attach resolver | Raw local paths and GitHub URLs are untrusted and must normalize into one safe canonical contract before later plans act on them. |
| `/attach` UI -> domain boundary | The LiveView must not grow ad hoc git or path-resolution logic that later diverges from execution code. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-01 | T | Local path parsing | mitigate | Canonicalize and validate local paths through one attach resolver; reject missing or non-repo paths with typed errors. |
| T-30-02 | S | GitHub URL identity | mitigate | Parse and normalize GitHub URLs into one owner/repo identity shape before any later workspace action. |
| T-30-03 | D | LiveView/domain drift | mitigate | Keep `/attach` thin and make it delegate to `Kiln.Attach`; lock behavior with focused unit and LiveView tests. |

</threat_model>
