<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| resolved source -> workspace root | The manager must not allow attached workspaces to escape the configured root or become ambiguous across sources. |
| workspace metadata -> later git workers | Persisted fields will later drive push / PR behavior, so they must be canonical and program-controlled. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-04 | T | Workspace path rooting | mitigate | Constrain all managed workspaces to a configured attach workspace root and prove it in tests. |
| T-30-05 | R | Repo identity persistence | mitigate | Persist canonical metadata via a narrow domain API, not raw user form fields, so later plans reuse stable repo facts. |

</threat_model>
