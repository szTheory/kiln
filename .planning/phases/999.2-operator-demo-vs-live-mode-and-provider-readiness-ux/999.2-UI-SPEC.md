# Phase 999.2 — UI Design Contract

> Visual and interaction contract for **operator demo vs live mode** and **provider readiness / connection UX**: a global mode signal in the shell, calm inline degradation when providers are unreachable or misconfigured in **live** mode, honest **demo** labeling, and a **config presence** readout (provider **names only** — **SEC-01**, never key material). Phoenix LiveView 1.1 + Tailwind v4. Extends **Phase 07** (`07-UI-SPEC.md`) and **Phase 08** (`08-UI-SPEC.md`); composes with `ProviderHealthLive` (`/providers`) and `ModelRegistry` health snapshots.

## Information Architecture (locked)

| Surface | Location | Notes |
|---------|----------|--------|
| Mode indicator | `Layouts.app` header row — **left of** factory summary band or **first cell** of a slim strip under the main nav | Always visible on authenticated operator routes; never blocks content |
| Provider readiness strip | Same header region **or** directly above `<main>` inside `Layouts.app` | Collapses to icon + short label on narrow viewports; full text from `md` breakpoint |
| Config presence | Sub-row or trailing segment of the readiness strip | Comma-separated **provider ids / display names** only; empty slot = "Not configured" per provider label pattern from Phase 08 |
| Deep diagnostics | Link target **`/providers`** only | No duplicate full health UI in the strip |
| `ProviderHealthLive` | `/providers` unchanged | Strip is **navigation + summary**, not a second dashboard |

## Spacing Scale

Inherit Phase 07 scale verbatim (multiples of **4px**):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip padding, icon-text gap inside mode pill |
| sm | 8px | Strip internal gap, pill-to-pill spacing |
| md | 16px | Strip padding block-axis, alignment with `FactoryHeader` |
| lg | 24px | Section separation if strip wraps to two lines |
| xl | 32px | Page gutters (unchanged) |
| 2xl | 48px | Rare — not used in chrome strip |
| 3xl | 64px | n/a in this phase |

Exceptions: **44px** minimum hit target for **icon-only** "dismiss for this session" control if added later (documented Phase 07 exception); prefer **labeled** control at `md` padding.

## Color

Explicit **60 / 30 / 10** (inherited roles):

| Role | Value | Usage |
|------|-------|-------|
| Dominant (~60%) | `#121212` (Coal) | Page background |
| Secondary (~30%) | `#1B1D21` (Char) / `#262B31` (Iron) | Header chrome, strip background, cards |
| Accent (~10%) | `#E07A3F` (Ember) | See reserved list |
| Destructive / severe | `#9A5634` (Clay) | Blocked-style provider state **border + label** only when run is blocked by provider (if surfaced here); not for normal "unreachable" calm state |
| Muted | `#8C857D` (Smoke) | Secondary strip copy |
| Readable | `#F5EFE6` (Bone) | Primary strip text |

**Accent (Ember) reserved for:** the **active mode pill outline or dot** (demo vs live), the **primary link** in the strip ("Open provider health" when degraded), and **one** in-viewport emphasis per strip — not every status word, not nav links.

**Provider status (semantic, not Ember fills):** use **border + text + optional `<.icon>`** aligned with `ProviderHealthLive` RAG: green/amber/red **tokens** as Tailwind utility classes consistent with existing cards (`border-emerald-700/70`, amber/warning, red/error) — **no** full Ember background blocks.

Accent reserved for: **mode indicator emphasis**, **single strip CTA link** to `/providers`, **focus ring** on interactive strip controls.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Live mode — chip title | "Live" |
| Live mode — supporting line (smoke, optional) | "Runtime credentials apply; external APIs may incur cost." |
| Demo mode — chip title | "Demo" |
| Demo mode — supporting line | "Outcomes use fixtures and stubs; no paid provider calls." |
| Live + provider unreachable (inline) | Title: "Provider not reachable" — Body: "Runs may stall until connectivity returns." — **Link:** "Open provider health" (`~p"/providers"`) |
| Live + credential not configured (`!key_configured?`) | Title: "Credential not configured" — Body: "This provider needs a configured secret reference in the runtime environment." — **Link:** "Open provider health" |
| Demo + simulated outage (if applicable) | Title: "Demo provider offline" — Body: "This state is simulated; switch to live mode for real connectivity." |
| Config presence — section label | "Providers" |
| Config presence — configured | "{Name}: configured" (one clause per provider, names only) |
| Config presence — not configured | "{Name}: not configured" |
| Strip empty / all healthy (live) | Omit strip **or** single line: "All configured providers operational" (12px label, smoke) — product choice: **prefer omit** to reduce noise; document in plan |
| Polling hint (if shown in strip) | Reuse ProviderHealthLive: "poll every {n}s" in mono, smoke |

**Primary CTA for this phase (strip):** "Open provider health" (never "Click here", never "OK").

**Empty state:** N/A for chrome strip; if a dedicated "runtime status" LiveView is added later, use: heading "No runtime issues detected", body "Provider health updates every few seconds. Open provider health for full detail."

**Error state (assign load failure for mode):** Heading: "Runtime mode unavailable" — Body: "The shell could not read demo vs live state. Open provider health to verify providers." — Link: "Open provider health"

**Destructive confirmation:** None in phase scope — mark **n/a** (no mode toggle that deletes data in v1 contract).

## Layout & Visual Hierarchy

| Surface | Primary focal | Behavior |
|---------|---------------|----------|
| `Layouts.app` chrome | **Mode chip** (left) then **readiness strip** (if any degradation) | Eye path: mode → problem summary → factory counts |
| Narrow viewport | Mode chip remains; strip compresses to **icon + one-line status** + chevron or link affordance | No horizontal scroll for copy |

**Accessibility:** Mode chip includes `aria-label` including mode name + short consequence ("Live mode, external APIs may be used"). Degradation strip uses `role="status"` and **does not** use `role="alert"` unless user action is required (prefer status for calm UX).

**Stable DOM ids:** `id="operator-mode-chip"`, `id="operator-provider-readiness"`, `id="operator-config-presence"`.
