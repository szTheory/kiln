- **Args:** `[run_id: uuid, limit: pos_integer, after: nil | {DateTime.t(), binary_id}]` (allow **`after: nil`** default).
- **Query:** `from e in Event, where: e.run_id == ^run_id`, apply keyset **`where: e.occurred_at > ^dt or (e.occurred_at == ^dt and e.id > ^id)`** when cursor set, **`order_by: [asc: e.occurred_at, asc: e.id]`**, **`limit(^limit + 1)`** internally OR pure **`limit`** with **`truncated: length==limit` AND next-page probe** — executor picks one **documented** approach; return **`truncated:`** true iff more rows exist forward.

- **Tests:** Insert **≥3** rows for a **`run_id`** with **crafted** `occurred_at` (use **`DateTime.add/3`** microseconds) so **two rows share** `occurred_at` but differ in **`id`**, proving **id** tie-break. Assert **`replay_page`** returns stable ordering.

- **Compatibility:** Existing **`replay/1`** must keep current call sites working.
