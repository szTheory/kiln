<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| attached_repos -> attach delivery | Persisted repo/workspace facts now drive branch, push, and PR behavior; drift or reparsing would target the wrong repo. |
| attach delivery -> managed workspace git state | Local branch creation mutates the workspace and must stay confined to the ready attached repo. |
| attach delivery -> external_operations workers | Push and PR creation cross from local state into external git/GitHub effects and must remain idempotent. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-31-01 | T | Branch-name derivation | mitigate | Validate sanitized branch names with `git check-ref-format --branch`, freeze once per run, and reuse the frozen value on retry. |
| T-31-02 | T | Attached-repo targeting | mitigate | Read `workspace_path`, `repo_slug`, `remote_url`, and `base_branch` only from `Kiln.Attach` persistence plus Phase 30 ready-state output, not raw operator input. |
| T-31-03 | R | External-op replay | mitigate | Reuse `PushWorker` and `OpenPRWorker` idempotency keys and frozen payloads so duplicate runs do not create extra pushes or PRs. |
| T-31-04 | E | Approval/workflow escalation drift | mitigate | Keep draft PR creation asynchronous and avoid adding a new human gate or blocked run state for attached repos. |

</threat_model>
