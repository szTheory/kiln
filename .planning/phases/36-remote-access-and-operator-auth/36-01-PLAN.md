<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → auth controller | untrusted requests enter the login/session surface |
| browser → LiveView socket | the authenticated scope is copied into socket assigns |
| Sigra auth hooks → Kiln.Audit | third-party auth events must not bypass the append-only ledger |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-01 | S/T | session cookies / login flow | mitigate | Use Sigra-issued signed session tokens, renew the Plug session on login, and keep the cookie httpOnly/same-site to reduce fixation and replay risk. |
| T-36-02 | I | login routes / router surface | mitigate | Keep signup/reset surfaces out of the router and make login responses enumeration-safe so operator existence is not leaked. |
| T-36-03 | E/T | auth audit path | mitigate | Forward Sigra auth events into `Kiln.Audit.append/1` only; do not create a second auth ledger or bypass the insert-only defenses. |
</threat_model>
