<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host → container tunnel | the tunnel process receives operator secrets and network access |
| compose profile → default compose | profile-gated config must not change the ordinary local startup path |
| tailnet → dashboard | the remote dashboard is reachable only through the private tailnet |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-04 | S | Tailscale sidecar | mitigate | Require `TS_AUTHKEY` from the operator's Tailscale admin console and keep the service profile-gated so only the intended tailnet can join. |
| T-36-05 | I | compose exposure | mitigate | Do not publish new public ports in the remote profile; route access through the private tailnet URL only. |
| T-36-06 | D/T | compose defaults | mitigate | Leave the default compose services untouched and make the remote tunnel strictly additive so local dev remains stable. |
</threat_model>
