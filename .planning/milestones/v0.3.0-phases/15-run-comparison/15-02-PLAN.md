  <action>
Add **`lib/kiln/runs/compare.ex`** with **`@moduledoc`** describing:

- Args: **`baseline_id :: String.t()`**, **`candidate_id :: String.t()`** (already validated UUID strings at LiveView boundary — still **`cast`** defensively or document precondition).
- Returns a **single struct** (define **`defstruct`** in same file, e.g. **`%Kiln.Runs.Compare.Snapshot{...}`**) containing at minimum:
  - **`:baseline_run`**, **`:candidate_run`** — `%Run{}` or **`nil`** if missing
  - **`:union_stage_ids`** — ordered list of **`workflow_stage_id`** strings (**union** of both sides)
  - **`:rows`** — enumerable of maps with keys **`workflow_stage_id`**, **`baseline_stage`**, **`candidate_stage`** (each **`%StageRun{}` or `nil`** for latest attempt per CONTEXT)
  - **`:artifact_rows`** — list of maps: **`logical_key`** (recommend **`name` + optional stage id disambiguation**), **`baseline_meta`**, **`candidate_meta`**, **`equality`** atom in **`[:same, :different, :unknown, :baseline_only, :candidate_only]`** derived from **`sha256`** presence
- **Queries:** at most **~4** DB round-trips: `runs` (batched `where id in ^ids` preferred), **`stage_runs`** per run (or single joined query if cleaner), **`artifacts`** per run with **`select: [:id, :name, :sha256, :size_bytes, :stage_run_id, :run_id, :content_type]`** only.
- **Ordering union keys:** Prefer **`Kiln.Workflows.graph_for_run/1`** order using **baseline** run when checksum matches on-disk workflow; **else** fall back to **alphabetical** `workflow_stage_id` — document in **`@moduledoc`**.

Delegate from **`Kiln.Runs`** with **`def compare_snapshot(baseline_id, candidate_id), do: Kiln.Runs.Compare.snapshot(baseline_id, candidate_id)`** **or** skip re-export if LiveView calls **`Compare`** directly — pick **one** pattern and use consistently in plan **03**.
  </action>
