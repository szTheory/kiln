  <action>
Create **`Kiln.Oban.PostMortemMaterializeWorker`** with **`use Kiln.Oban.BaseWorker, queue: :default`**.

Args (string keys): **`"run_id"`**, **`"idempotency_key"`** where key is exactly **`"post_mortem_materialize:" <> run_id`**.

**`perform/1` flow:**

1. Call **`fetch_or_record_intent/2`** with **`op_kind: "post_mortem_materialize"`** and payload including **`run_id`**.
2. If op already **`:completed`**, return **`:ok`**.
3. Load **`stage_runs`** for **`run_id`** ordered; compute **`source_watermark`** as **`max(occurred_at)`** from **`audit_events`** for **`run_id`** (UTC ISO8601 string) **plus** optional **`max_audit_id`** UUID string for forensics — **`audit_events.id` is `:binary_id`**, not integer.
4. Build **`snapshot` map** with at least keys: **`"stages"`** (list of per-stage maps with **`workflow_stage_id`**, **`requested_model`**, **`actual_model_used`** if columns exist — grep **`stage_runs`** schema), **`"source_watermark"`**, **`"generated_at"`** ISO8601.
5. Call **`Kiln.Runs.PostMortems.upsert_snapshot/2`** setting **`status: :complete`** on success.
6. Optionally **`Kiln.Audit.append(%{event_kind: :post_mortem_snapshot_stored, ...})`** with schema-conformant payload.
7. **`complete_op/2`** with minimal result map.

**PII:** Do **not** embed raw prompts or tool payloads (D-1906).
