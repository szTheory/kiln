# Milestone v0.8.0 — Requirements

## Goal
Transition Kiln from a local-only tool into a remote-capable factory that can autonomously merge, release, and verify deployments.

## Requirements

### Remote Access & Security
- [x] **REMOTE-01**: Secure the Phoenix dashboard with single-operator Auth (Passkey or Password) for safe remote visibility. (Validated in Phase 36)
- [x] **REMOTE-02**: Provide a `remote` Compose profile with a sidecar tunnel (Tailscale/Cloudflare) for mobile/laptop connectivity. (Validated in Phase 36)
- [x] **REMOTE-03**: Dashboard remains read-only for unauthenticated users (if any public surface exists), with full operator capabilities locked behind Auth. (Validated in Phase 36)

### Autonomous Release & Versioning
- **DELIVER-01**: Kiln autonomously bumps versions and generates changelogs using Conventional Commits across attached-repo runs.
- **DELIVER-02**: Kiln supports binary/library releases via `gh release` and registry publishers (`mix hex.publish`, `npm publish`, etc.).
- **DELIVER-03**: Release operations are idempotent and tracked via `external_operations` to prevent double-publishing.

### SaaS Delivery & SRE
- **DELIVER-04**: Kiln supports SaaS deployment (Fly.io/Cloudflare) and surfaces a clickable live URL on the run dashboard.
- **DELIVER-05**: Post-deploy SRE smoke check: Kiln verifies health of the deployed service (HTTP 200) and escalates on failure.
- **DELIVER-06**: Deploy tokens and registry secrets are managed via `persistent_term` + ENV, never stored in plaintext in the database.

### Verification (UAT)
- **UAT-07**: End-to-end "closed loop" proof: A run against an attached repo successfully merges code, bumps the version, creates a GitHub Release, deploys to a SaaS provider, and passes a post-deploy health check.
