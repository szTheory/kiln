## Mermaid (DOCS-04)

- **Risk:** Client-only Mermaid passes build but ships broken diagrams.
- **Mitigation paths:** (a) `@astrojs/mdx` + rehype pipeline that validates at build, or (b) CI step running `mmdc` / `npx @mermaid-js/mermaid-cli` over extracted fences on `site/**` changes.
- **Pin:** `mermaid` and CLI in lockfile; document upgrade checklist (CONTEXT D-9991-32).

## Validation Architecture

### Dimension map (execution feedback)

| Dimension | Signal | Tool / artifact |
|-----------|--------|-----------------|
| Build integrity | Site compiles | `pnpm exec astro build` in CI |
| Internal navigation | No broken anchors | `htmltest` on `dist/` |
| External references | No unexpected 4xx/5xx on checked URLs | `lychee` with curated excludes |
| Prose quality | Typos caught | `typos` |
| Diagram syntax | Mermaid blocks valid | Build-time plugin OR `mmdc` CI step |
| Publish | Pages receives artifact | `deploy-pages` success on `main` |

### Nyquist sampling (per-plan)

- After **scaffold plan:** `pnpm install && pnpm exec astro build` locally or in CI job for that PR.
- After **CI plan:** full docs workflow file passes `actionlint` (optional) + workflow dry-run via PR.
- Before phase sign-off: merge to `main` produces green docs workflow + live Pages URL (manual spot-check allowed for DNS/cache).

### Wave 0

- No separate test framework — **static site + CLI tools** constitute test infrastructure. First plan establishes `site/package.json` + lockfile as the foundation.
