## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local machine -> readiness probes | `gh auth status`, `docker info`, and runtime env checks read untrusted machine state |
| backend summary -> LiveView UI | operator-facing readiness data crosses from server logic into rendered remediation copy |
| operator click -> verify actions | verify buttons trigger server-side probe/update actions |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-25-02 | T | lib/kiln/operator_setup.ex | mitigate | Keep provider/config output names-only per D-2504 and SEC-01; do not expose secret values or raw env contents. |
| T-25-03 | I | lib/kiln_web/live/settings_live.ex | mitigate | Render only checklist metadata, provider names, and remediation guidance; no secret refs or credential values in the UI. |
