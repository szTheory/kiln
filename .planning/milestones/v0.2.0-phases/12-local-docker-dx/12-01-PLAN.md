# Phase 12 Plan 01 — Local Docker / dev environment DX

## Objective

Pick **one** optional strategy so a new operator can go from zero to a running Kiln with fewer footguns: **devcontainer** (VS Code / Cursor), **Compose `app` service** wrapping `mix phx.server`, or a **`make` / `just` documented wrapper** — and ship the minimum files + README section so the path is reproducible.

## Preconditions

- Phase **10** complete (runbook and baseline host+Compose path are accurate per `LOCAL-DX-AUDIT.md`).

## Falsifiable UAT

1. A second machine (or clean clone directory) follows **only** the new doc path and reaches `/onboarding` without undocumented steps.
2. The chosen strategy is **one** primary path (no three parallel “official” ways).
3. `mix check` still passes; no Docker socket mount in any sandbox or dev path (project invariant).

## Tasks (outline)

1. Spike: compare devcontainer vs Compose app for this repo’s asset pipeline + DB role (`kiln_owner` migrations).
2. Implement the chosen option; delete or demote alternatives to “advanced”.
3. Update `README.md` + link from `LOCAL-DX-AUDIT.md`.

## Out of scope

- Production cloud deploy.
- Replacing host-based dev entirely if the team still prefers local Elixir — document both “full container” and “host app” only if Phase 10 already labels the default.
