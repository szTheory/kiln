# Phase 10 Plan 01 — Local operator readiness

## Objective

Close the gap between “clone exists” and “operator can drive Kiln from LiveView with confidence”: documented path, env checklist, and a **dry-run narrative** that matches `compose.yaml` + host `mix phx.server` (see `.planning/research/LOCAL-DX-AUDIT.md`).

## Falsifiable UAT (layered — Phase 9 D-930..D-934, Phase 10 CONTEXT)

**Machine / integration (authoritative script):**

1. **Fresh env simulation** — From a clean shell (no stale `MIX_ENV`), with only `.env` from `.env.sample` filled (`SECRET_KEY_BASE`, DB URL, `PORT`), `docker compose up -d db` healthy, `KILN_DB_ROLE=kiln_owner mix ecto.migrate` (or `mix setup`) succeeds.
2. **Boot + health** — `bash test/integration/first_run.sh` completes: migrates, boots `mix phx.server`, asserts **`/health` JSON** (dependency fields green). This is the **single SSOT** integration smoke — do not duplicate orchestration in Mix unless adding a one-line alias that delegates to this script.
3. **Boot invariant** — `mix phx.server` starts without `KILN_SKIP_BOOTCHECKS` when driven manually the same way as README (BootChecks green).

**Browser / LiveView (human + test suite — not `first_run.sh`):**

4. **Onboarding** — Operator: `http://localhost:4000/onboarding` loads; wizard can complete **or** surfaces typed blockers for missing keys / Docker as designed. **Automated proof** remains existing **`Phoenix.LiveViewTest`** onboarding tests — do not claim `first_run.sh` validates the wizard (D-934).
5. **Run board** — After onboarding, `/` shows the run board (empty state acceptable).

**Optional observability (documented + manual once, not default `mix check`):**

6. **Optional traces** — Follow README **Traces (local)**: `docker compose up -d otel-collector jaeger`, set `OTEL_EXPORTER_OTLP_ENDPOINT`, perform one **documented** trivial action (e.g. load `/health` or `/onboarding` in browser), confirm a span in Jaeger at `http://localhost:16686`. Record the chosen path and result in **10-01-SUMMARY.md** when the plan completes. **Do not** add Jaeger to default CI for this phase.
