# Markdown spec stub

Example **markdown-first** spec for operators who paste requirements from docs or tickets.

## Summary

- **Problem:** Describe the user-visible defect or missing capability in one paragraph.
- **Constraints:** List hard limits (time, files, providers, secrets).
- **Definition of done:** Bullet list of observable outcomes.

## Test plan

- Unit tests for new pure functions.
- LiveView or integration tests when UI changes.

## Rollout

- Feature-flagged or behind env only if explicitly required.
